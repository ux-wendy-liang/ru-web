## 一、当前问题诊断

### 1. 首页 (Home) 问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **缺少影响力前置** | 高 | 项目卡片只显示描述，没有展示关键成果数据（如 +45% 转化率） |
| **项目标题过长** | 中 | "AI Coach Mentor - AI-Powered Coaching Practice Platform" 太长，不够精炼 |
| **缺少项目标签** | 中 | 没有标签帮助快速识别项目类型（如 "0→1 产品"、"B2B"、"移动端"） |
| **Hero 区域缺少个人照片** | 中 | 顶级设计师的 Portfolio 通常在首页就展示个人形象，增加亲切感 |
| **Experience 部分缺少 logo** | 低 | 公司 logo 能增加可信度 |

### 2. 项目详情页 (ProjectDetail) 问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **影响力没有前置** | 高 | Impact 放在页面底部，应该放在顶部让招聘经理一眼看到 |
| **缺少"用户的故事"** | 高 | 按指南要求，应该用用户视角讲故事，而不是"我做了什么" |
| **Challenge 标题处理有 bug** | 中 | `project.challenge.split(':')[1]` 这种处理方式不够优雅 |
| **没有 Before/After 对比** | 中 | 对比图是最直观展示设计价值的方式 |
| **图片缺少说明文字** | 中 | 每张图应该有简短说明，帮助理解 |
| **没有下一个项目导航** | 低 | 应该有"查看下一个项目"引导用户继续浏览 |

### 3. 关于页 (About) 问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **缺少职业经历时间线** | 中 | 应该有清晰的职业发展路径展示 |
| **个人兴趣与设计能力关联弱** | 低 | Coaching 技能可以更好地与 UX Research 关联 |
| **缺少联系方式 section** | 低 | About 页底部应该有明确的联系入口 |

### 4. 内容问题（基于 Case Study Storytelling 指南）

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **以"我"为中心，而非用户** | 高 | 指南强调"用户是主角"，但当前内容多是"I designed"、"I proposed" |
| **缺少用户痛苦的具体描述** | 高 | 没有让读者感受到用户的挫败感 |
| **缺少转折点/洞察** | 高 | 没有那个"啊哈"时刻，没有讲清楚发现了什么关键 insight |
| **Impact 不够量化** | 中 | 有些项目的 impact 是定性的，缺少具体数字 |
| **缺少用户引用/反馈** | 中 | 真实的用户语言能大大增加可信度 |

### 5. 视觉设计问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **项目卡片缺少 hover 效果** | 低 | 可以添加图片放大或 overlay 效果 |
| **缺少页面过渡动画** | 低 | subtle 的动画能提升体验 |
| **移动端体验未优化** | 需测试 | 需要在真机上测试 |

## 二、优化计划

### Phase 1: 内容重写（优先级：最高）

**目标**: 按 Case Study Storytelling 指南重写所有项目内容

#### 1.1 重构 projects.ts 数据结构

```typescript
// 新增字段
interface Project {
  // ... 现有字段 ...

  // 新增：影响力前置
  impactMetrics: {
    value: string;      // 如 "+45%"
    label: string;      // 如 "Filter engagement"
  }[];

  // 新增：用户故事结构
  userWorld: string;           // 用户是谁？在什么场景？
  userPain: string;            // 用户的痛苦是什么？
  userQuote?: string;          // 用户原话（增加真实感）
  stakes: string;              // 问题有多严重？影响多少人/钱？
  insight: string;             // 关键洞察/转折点
  beforeAfter?: {              // 前后对比
    before: string;
    after: string;
  };

  // 新增：图片说明
  imageDescriptions?: string[];  // 每张图的说明文字
}
```

#### 1.2 按指南重写每个项目内容

**示例：Publisher Portal 项目改写**

**现在的写法（以"我"为中心）**:
> "Proposed and convinced to expand the scope from a feature level to become an entire new publisher portal."

**改写后（以用户为中心）**:
> "Publishers struggled to integrate and manage their LinkedIn Ads SDK—a process that was fragmented, confusing, and unsupported. Without a centralized hub, publishers spent hours navigating scattered documentation and troubleshooting integration issues alone."

### Phase 2: 项目详情页重构（优先级：高）

**目标**: 影响力前置 + 用户故事结构

#### 2.1 新的页面结构

```
1. 项目快照（顶部）
   - Role | Timeline | Team

2. 影响力展示（前置！⭐）

   - 用户反馈引用

3. 用户的世界（背景）
   - 用户是谁？
   - 他们想做什么？

4. 用户的痛苦（问题）
   - 遇到什么困难？
   - 用户原话引用

5. 问题的严重性（stakes）
   - 影响多少人/钱？
   - 为什么必须解决？

6. 转折点（洞察）
   - 发现了什么关键 insight？
   - Before/After 对比

7. 解决方案
   - 设计图片（带说明）
   - 交互演示 GIF

8. 结果
   - 重申 impact 数据
   - 用户反馈

9. 反思（可选）
   - 学到了什么？

10. 下一个项目导航
```

## 三、重写示例：Publisher Portal

### 现有内容

```
title: 'Building the LinkedIn Publisher Portal from 0-1'
shortDescription: 'Proposed and convinced to expand the scope from a feature level to become an entire new publisher portal.'
challenge: 'Privacy policy changes from Google Chrome and Apple disrupted digital advertising. LinkedIn faced losing 23-30% of LinkedIn's ad revenue.'
solution: 'Rather than simply adding an SDK integration feature, I proposed a comprehensive Publisher Portal—a dedicated hub where publishers could integrate, manage, and optimize their LinkedIn Ads SDK efficiently.'
impact: [
  'Scope expansion from minor feature to full product platform',
  'Protected 23-30% of ad revenue',
  'Established foundation for long-term publisher ecosystem'
]
```

### 按指南重写

```typescript
{
  id: 'publisher',
  title: 'LinkedIn Publisher Portal',  // 简化标题
  subtitle: 'Building a 0→1 platform that protected $XX million in ad revenue',
  year: 2024,

  // 影响力前置（大号数字）
  impactMetrics: [
    { value: '23-30%', label: 'Revenue protected' },
    { value: '0→1', label: 'New product platform' },
    { value: '6 weeks', label: 'From concept to launch' }
  ],

  // 用户的世界
  userWorld: 'Digital publishers—news sites, blogs, content platforms—rely on advertising revenue to sustain their businesses. They need to integrate ad SDKs to monetize their content, but the process is often fragmented and frustrating.',

  // 用户的痛苦
  userPain: 'When privacy policy changes from Chrome and Apple disrupted ad tracking, publishers suddenly faced a crisis. They needed to quickly adopt LinkedIn\'s new SDK, but there was no clear path forward—just scattered documentation and zero support.',

  // 用户原话
  userQuote: '"I spent 3 days just trying to figure out how to integrate the SDK. There was no dashboard, no status updates, nothing."',

  // 问题严重性
  stakes: 'This wasn\'t just a UX inconvenience—it was an existential threat. If publishers couldn\'t integrate smoothly, LinkedIn risked losing 23-30% of its ad revenue. That\'s potentially hundreds of millions of dollars.',

  // 转折点（洞察）
  insight: 'Through user interviews, I discovered something critical: publishers didn\'t just need SDK documentation—they needed a command center. They wanted to see integration status, manage team access, monitor performance, and troubleshoot issues all in one place. The ask for "SDK integration" was actually a need for "publisher operations platform."',

  // Before/After
  beforeAfter: {
    before: 'Scattered documentation, no visibility into integration status, manual support requests',
    after: 'Centralized portal with real-time status, self-serve troubleshooting, team management'
  },

  // Solution 保持原有图片，但添加说明
  imageDescriptions: [
    'Publisher journey map showing 7 key touchpoints',
    'User persona: the overworked publisher ops manager',
    'Competitive analysis: how Meta and Google handle publisher onboarding',
    // ...
  ],

  // 重新组织的 Impact
  impact: [
    'Expanded scope: Convinced stakeholders to build a full platform instead of a simple feature—demonstrating strategic product thinking',
    'Protected revenue: The portal enabled smooth SDK adoption, safeguarding 23-30% of LinkedIn\'s ad revenue',
    'Future-proofed: Created the foundation for LinkedIn\'s publisher ecosystem, enabling future monetization features'
  ],

  // 反思
  learnings: 'This project taught me the power of reframing. The original ask was small—"add SDK integration." But by deeply understanding user needs, I uncovered a much larger opportunity. Sometimes the best design decision is expanding the problem space.'
}
```

## 五、检查清单（完成后对照）

### 内容检查（基于 Case Study Storytelling 指南）

- [ ] 读者能清楚感受到用户之前的痛苦吗？
- [ ] 读者能理解这个问题为什么重要（业务影响）吗？
- [ ] 有没有清晰的"啊哈"时刻（关键洞察）？
- [ ] Before/After 对比明显吗？
- [ ] 有数据证明结果吗？
- [ ] 全文"我"出现的次数是否太多？
- [ ] 读完后，读者能自己得出"这个设计师很厉害"的结论吗？

### 视觉检查（基于顶级 Portfolio 研究）

- [ ] 影响力是否前置（60秒内能看到关键数据）？
- [ ] 是否有用户引用/反馈？
- [ ] 项目数量是否精简（2-3个深度 > 10个浅层）？
- [ ] 移动端体验是否流畅？
- [ ] 图片是否都有说明文字？
